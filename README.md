# m4-ComfyUI

[日本語版 (Japanese Version)](README_ja.md)

A collection of custom nodes for ComfyUI. These nodes are designed to streamline workflows, such as caption creation for LoRA training.

## Installation

1. Navigate to the `custom_nodes` directory in your ComfyUI installation.
2. Clone this repository:
   ```bash
   git clone https://github.com/m4logicworks/m4-ComfyUI.git
   ```
3. Restart ComfyUI.

## Node List

### 1. M4 Sequence Image Loader
Loads images from a specified folder sequentially in alphabetical order. This node is useful for batch processing, allowing you to process images in a folder one by one.

- **Category**: `M4/loaders`
- **Inputs**:
  - `folder_path` (STRING): Absolute path to the folder containing images.
  - `image_index` (INT): Index of the image to load (0-indexed). Connecting a `Primitive` node and setting it to `increment` allows you to load images sequentially.
- **Outputs**:
  - `IMAGE`: Loaded image data (automatically auto-rotates based on Exif information).
  - `STRING`: Filename without extension (e.g., `001` for `001.png`).
  - `INT`: Image Width.
  - `INT`: Image Height.

### 2. M4 Save Text
Saves text data (e.g., captions generated by an LLM) to a specified folder as a text file (`.txt`).

- **Category**: `M4/savers`
- **Inputs**:
  - `text` (STRING): The text data to save.
  - `folder_path` (STRING): Absolute path to the folder where the text file will be saved.
  - `filename` (STRING): Filename to save (the `.txt` extension is added automatically).
  - `collision_behavior` (COMBO): Behavior when a file with the same name already exists:
    - `replace`: Overwrites the existing file (default).
    - `rename`: Appends a sequential number to the filename (e.g., `output_1.txt`).
- **Outputs**:
  - `STRING`: Absolute path to the saved file.

### 3. M4 Prompt Node
Manages text inputs and allows dynamic selection and loading of prompts from a specified JSON file.

- **Category**: `M4/text`
- **Inputs**:
  - `text_1` (STRING, multiline): Any text input.
  - `json_file_path` (STRING): Path to the prompt JSON file.
  - `prompt_selector` (COMBO): List of keys loaded dynamically from the JSON file to select from.
  - `text_2` (STRING, multiline): Loaded prompt text corresponding to `prompt_selector` (automatically filled but can be edited manually).
  - `output_choice` (COMBO): Source of the output. Select either `Text 1` or `Text 2`.
- **Outputs**:
  - `text` (STRING): The text data from the selected source.

### 4. M4 Resolution Preset Node
Outputs Width and Height based on a selected resolution preset for image generation or video streaming.

- **Category**: `M4/utils`
- **Inputs**:
  - `preset` (COMBO): Resolution presets including Square (1024, 1536, 2048), SDXL/Flux, 2K resolutions, HD/FHD video standards, etc. Choose `Custom` to set custom dimensions.
  - `custom_width` (INT): Custom width output when `Custom` preset is selected (default: 1024).
  - `custom_height` (INT): Custom height output when `Custom` preset is selected (default: 1024).
  - `invert_aspect` (BOOLEAN): If `True`, swaps width and height (useful for toggling between portrait and landscape).
  - `batch_size` (INT): Batch size (default: 1).
- **Outputs**:
  - `width` (INT): Output width in pixels.
  - `height` (INT): Output height in pixels.
  - `batch_size` (INT): Set batch size.

### 5. M4 Texture Generator
Generates various textures and noise images to be used as base materials for image generation (e.g., i2i noise injection).

- **Category**: `M4/image`
- **Inputs**:
  - `width` (INT): Width of the generated texture (default: 1024).
  - `height` (INT): Height of the generated texture (default: 1024).
  - `batch_size` (INT): Batch size (default: 1).
  - `texture_type` (COMBO): Type of texture to generate (12 options):
    - `White Noise`, `Gaussian Noise`, `Film Grain`, `Perlin Noise (Smooth)`, `Fractal Noise (High Detail)`, `Canvas Texture`, `Paper Texture`, `Horizontal Brush`, `Vertical Brush`, `Diagonal Hatching`, `Vignette`, `Color Cloud`
  - `scale` (FLOAT): Scale of the noise/texture (default: 1.0).
  - `strength` (FLOAT): Apply strength over the base color. 0.0 means solid color, 1.0 means full texture (default: 0.5).
  - `color_preset` (COMBO): Preset for the background color (White, Black, Gray, Red, Green, Blue, Sepia, Warm, Cool, Custom HEX).
  - `custom_hex` (STRING): Custom HEX color code used when `Custom HEX` is selected (default: `#FFFFFF`).
- **Outputs**:
  - `IMAGE`: Generated RGB image.

### 6. M4 Wildcard Loader
Loads text data from YAML wildcard files and outputs one item chosen randomly, sequentially, or by a fixed index.

- **Category**: `M4/text`
- **Inputs**:
  - `wildcard_dir` (STRING): Absolute path to the folder containing wildcard files.
  - `wildcard_selector` (COMBO): Wildcard keys scanned automatically from YAML files in the directory (e.g., `sample/gravure_pose/basic_pose`).
  - `preview_text` (STRING, multiline): Preview of the items available in the selected wildcard (read-only).
  - `mode` (COMBO): Selection mode:
    - `random`: Reproducible random item selection based on the `seed`.
    - `sequential`: Selects items in order per execution (Queue Prompt), wrapping back to the start when the end is reached.
    - `fixed`: Selects the item at the index specified by `fixed_index`.
  - `fixed_index` (INT): The index of the item to select in `fixed` mode (0-indexed).
  - `seed` (INT): Random seed used in `random` mode.
- **Outputs**:
  - `text` (STRING): Selected text data.

#### YAML Wildcard Format
Supports `.yaml` / `.yml` files with hierarchical structures (dictionaries and lists) as follows:

```yaml
gravure_pose:
  basic_pose:
    - standing, cowboy shot, looking at viewer
    - standing, cowboy shot, arms behind head
  lying_pose:
    - lying on stomach, looking at viewer
    - lying on side, looking at viewer
```

This node allows you to select not only the leaf list items (e.g., `basic_pose`) but also intermediate categories (e.g., `gravure_pose`) or file-level root keys. Selecting a higher-level category will merge all list items under it.
